## My Teaching Style

I'm not going to give you code to memorize.

Instead, I'll:

1. Explain the concept.
2. Build it with you.
3. Remove parts and ask you to finish them.
4. Introduce bugs for you to fix.
5. Gradually reduce guidance until you're designing features yourself.

That way, you learn to think like an engineer rather than copy code.
